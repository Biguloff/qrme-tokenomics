Как работает сейчас (MVP)

iOS напрямую вызывает OpenRouter API. Нет бэкенда, нет списания монет, API ключ захардкожен.

Step 1 · iOS
Пользователь вводит промпт
Кнопка «✦ Изменить с ИИ ⬡5» → overlay с полем «Что изменить?» → «Изменить»
↓
Step 2 · iOS
Подготовка фото
Resize до 1024px → JPEG 0.75 → Base64 (~200-300KB)
↓
Step 3 · OpenRouter временно
POST openrouter.ai/api/v1/chat/completions
Model: google/gemini-2.5-flash-image · Multimodal: base64 image + text · API key в Swift — небезопасно
↓
Step 4
Результат: base64 PNG
message.images[0].image_url.url → decode → UIImage → редактор
Проблемы: API key в коде iOS · Монеты не списываются · Нет rate limit · OpenRouter — лишний посредник · Нет логов

Почему не aggai.ru?

input принимает только строку. При массиве content — ошибка input.substring is not a function. Без фото в input модель генерирует новое изображение вместо редактирования.

Как должно работать в production

Всё через бэкенд QRMe + aggai.ru. Никаких ключей в клиенте.

Step 1 · iOS
Промпт + фото
Resize 1024px → JPEG → base64 → POST /v3/ai/edit-image (JWT auth)
↓
Step 2 · Backend сделать
POST /v3/ai/edit-image
JWT → userId → проверить баланс ≥ 5 монет → rate limit 10 req/min
↓
Step 3 · aggai.ru multimodal
apiConversationMessages + content массив
"content": [{"type":"image_url","image_url":{"url":"data:image/jpeg;base64,..."}}, {"type":"text","text":"Edit..."}]
↓
Step 4 · Backend
Обработка
Получить base64 → списать 5 монет → записать usage → вернуть {resultImageBase64, coinsUsed, coinsRemaining}
↓
Step 5 · iOS
Результат
Decode base64 → UIImage → imageView
Преимущества: Ключи на сервере · Реальное списание монет · Rate limit · Usage log · Единая точка через aggai.ru

Платёжный шлюз и монеты

Три платёжные системы уже реализованы. Платёжный шлюз внутри QRMe нужно подключить. Оплата через ООО «КуарМи». Потоки денег через aggai.ru.

Таблицы БД

CREATE TABLE user_coins (
  user_id UUID PRIMARY KEY REFERENCES users(identifier),
  balance INTEGER NOT NULL DEFAULT 0,
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE coin_transactions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(identifier),
  amount INTEGER NOT NULL,        -- + пополнение, − списание
  type VARCHAR(20) NOT NULL,      -- 'topup','ai_edit','ai_chat','ai_image_gen'
  description TEXT,
  metadata JSONB,                 -- {model, prompt_preview, cost_usd, tokens, payment_id}
  created_at TIMESTAMP DEFAULT NOW()
);
