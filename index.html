Как считается. Внутренний курс для cost-basis: 1$ = 100 ₽. Наценка ×2 на cost → пользователь платит 200 монет за каждый 1$ реальной стоимости модели. В среднем это даёт 50–55% маржи для нас. Реальный курс $→₽ (≈ 90 ₽/$) добавляет ещё ~5% маржи сверху.
Статус. iOS-часть готова, смержена в main (mobile/ios). Subscription UI работает на локальных моках — юзер видит полный UX с 4 тирами. Формула

coins_to_charge = ceil( aggai_usd_cost × 200 )

// где 200 = 100 (внутренний $→₽) × 2 (наценка на cost)
// ceil() обеспечивает минимум 50% маржи — округление всегда в нашу пользу

// Пример — Gemini 3.1 Pro chat:
// aggai charges us $0.0372 per message
// 0.0372 × 200 = 7.44 → ceil = 8 монет
// Cost to us: 0.0372 × 100₽ = 3.72 ₽
// Revenue: 8 × 1₽ = 8 ₽
// Profit: 8 - 3.72 = 4.28 ₽ (53.5%)
